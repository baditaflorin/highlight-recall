# Phase 3 Input Audit

Status key: green = works end-to-end, yellow = partial, red = claimed/built but broken for real users, gray = not built and not claimed.

| Input pathway             | Status before | Evidence                                                                                                                               | User impact                                                                    | Phase 3 decision                                                    |
| ------------------------- | ------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------- |
| File picker: EPUB         | yellow        | `.epub` accepted and parsed through JSZip, but errors bubble as raw failures and some valid EPUB media types are skipped.              | Users can import simple EPUBs but get stuck on malformed or nonstandard files. | Keep and harden with per-file errors.                               |
| File picker: PDF          | yellow        | PDF.js text extraction works for text PDFs, but scanned PDFs produce zero highlights without a domain explanation.                     | Users think import failed silently on scanned/annotation-heavy PDFs.           | Keep and explain zero-text outcomes.                                |
| File picker: TXT/Markdown | green         | Plain file import works.                                                                                                               | Good base path for manual exports, notes, and copied highlights.               | Keep and route via format sniffing.                                 |
| Multi-file upload         | yellow        | `multiple` is enabled, but one thrown error stops the batch and there is no per-file summary.                                          | One bad file can hide successes and force rework.                              | Finish with partial success accounting.                             |
| Drag and drop             | gray          | No drop handlers.                                                                                                                      | Desktop users naturally drop files and nothing happens.                        | Add as same pipeline as file picker.                                |
| Paste text                | yellow        | Manual textarea accepts pasted highlight text, but app does not handle paste events as imports and does not parse multiple highlights. | Users paste blocks and get one giant card.                                     | Finish by parsing pasted text/HTML into candidates.                 |
| Paste HTML                | gray          | No HTML paste handling.                                                                                                                | Users copying web article excerpts lose structure and cleanup.                 | Add text extraction from clipboard HTML.                            |
| Clipboard read button     | gray          | No button or permission handling.                                                                                                      | Users must focus the textarea and paste manually.                              | Add explicit clipboard import with fallback copy.                   |
| URL import                | gray          | No URL field.                                                                                                                          | Browser CORS makes direct fetch unreliable.                                    | Keep out of scope; document paste-rendered-text guidance.           |
| Mobile picker             | yellow        | Native file input should work, but copy mentions only desktop formats and no mobile guidance.                                          | Mobile users can choose files but may not understand expected path.            | Keep file input; add `capture` only not relevant for reading files. |
| Folder import             | gray          | No directory input.                                                                                                                    | Nice-to-have for large libraries, not v1 claimed.                              | Out of scope.                                                       |
| Sample/demo loader        | gray          | Demo screenshot only; no sample state.                                                                                                 | First-time users have to create data before understanding output.              | Add a sample loader as an input path.                               |
| Imported state restore    | red           | JSON export exists but no JSON import.                                                                                                 | Backups are one-way; users cannot move browsers.                               | Finish with versioned state import.                                 |
| Autosave restore          | green         | IndexedDB restores documents/highlights on reload.                                                                                     | Core local-first behavior works.                                               | Keep and add migration policy.                                      |
| Deep links/share state    | gray          | Not built.                                                                                                                             | Not necessary for personal reading library v1.                                 | Out of scope except small copied text.                              |

Before counts: green 2, yellow 5, red 1, gray 7.
